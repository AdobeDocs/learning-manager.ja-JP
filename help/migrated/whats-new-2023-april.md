# このリリースの新機能（2023年4月）

## Adobe版Learning Manager Microsoft Teams版

Microsoft Teams向けの新しいAdobeのLearning Managerアプリは、業務フローの中で学習を促進し、ソーシャルラーニングを強化するように設計されています。 学習者は、ブラウザーに切り替えることなく、Microsoft Teamsプラットフォーム内の学習コンテンツにアクセスできるようになります。 MS Teams上のAdobe Learning Managerアプリのベータリリースについては、CSAMにお問い合わせください。

詳しくは、「 [Adobe版Learning Manager Microsoft Teams版](/help/migrated/adobe-learning-manager-app-microsoft-teams.md).

## インストラクターによるトレーニング(ILT)体験の機能強化

インストラクターによるトレーニング(ILT)体験にいくつかの機能強化が行われました。 主な機能強化には、場所に基づいてセッションをフィルタリングする機能、進行状況を失うことなくインスタンスを切り替える機能(VILT)、予約セッションの競合を管理する新しい「スケジュールアシスタント」、インストラクターに「スキル」を添付する機能、スキルに基づいてインストラクターを選択する機能などがあります。

### 変更点

* 管理者アプリの「スキル」ページに、スキルインストラクターのマッピングをアップロードするオプションがあります。
* 「スキル」ページに「インストラクター」という新しい列が追加されました。 列には、スキルに対するインストラクターの数が表示されます。 「インストラクター」列の番号をクリックすると、ポップアップにリダイレクトされます。ポップアップには、スキルに割り当てられたインストラクターの数が表示されます。

詳しくは、「 [インストラクターへのスキルの割り当て](/help/migrated/administrators/feature-summary/skills-levels.md#assign-skills-to-instructors).

![インストラクターのCSV](assets/instructor-csv-new.png)

### スケジュールアシスタント

予約インストラクターとクラスルームまたはバーチャルクラスルームの競合を管理します。 インストラクターをコースに割り当てる前に、インストラクターが対応可能な日時を知りたい場合は、スケジュール設定アシスタントを使用します。

作成者、管理者、カスタム管理者/作成者は、スケジューリングアシスタントを使用できます。

詳しくは、「 [スケジュールアシスタント](/help/migrated/authors/feature-summary/courses.md#scheduling-assistant).

## 新しいAdobeのLearning Managerモバイルアプリ

AndroidおよびiOS向けの全く新しいAdobeのLearning Managerモバイルアプリにより、外出先でも学習者がスムーズに学習にアクセスできます。 このアプリを使用すると、学習者はモバイルデバイスでトレーニングを受講し、後でデスクトップから再開できます。 通知、ファイルのアップロード、ブックマークの容易さ、学習コンテンツの共有、ソーシャル学習などの機能をサポートすることで、モバイルアプリは学習者に柔軟性を提供し、ジャストインタイムの学習をサポートします。

![新しいモバイルマップインターフェイス画像](assets/mobile-app.png)

Google Play StoreおよびApple App Storeからモバイルをダウンロードします。 アプリの古いバージョンのユーザーには、新しいアプリをダウンロードしてインストールするように求められます。

このリリースでは、アプリは次の機能をサポートしています。

このリリースでは、新しいモバイルアプリが次の主な機能をサポートしています。

* 学習者の最近の検索や組織内の一般的な検索を検索入力ボックスに表示するなど、検索が強化されました。
* 学習者のトランスクリプトをダウンロードできます。
* 通知センターの簡易化 – アプリ内の通知として、学習者に重要な更新を提供します。
* カタログや学習状況ページで学習目標を並べ替えたり、フィルタリングしたりする機能
* 学習目標(LO)をブックマークする機能 – ブックマークされたLOは、学習者ホームページの「自分が保存」セクションで利用できます。
* 拡張LPのサポート
* モバイルインターフェイスの言語を、Learning Managerがサポートする任意のAdobeに変更することができます。
* 社外の資格認定、CR/VCセッション、およびアクティビティモジュールでのファイルのアップロードのサポート
* アプリ内でL1コースフィードバックを送信できます。
* ディープリンクのサポート。
* ソーシャル学習のサポート
* バッジのサポート。
* 外部ユーザーログインのサポート。

**このリリースでサポートされていない機能**

* モバイルアプリ内からコースを取得したときに、コース内のクイズモジュールへの複数回の試行をサポートします。
* コースのメタデータ、タグ、検索範囲を指定するスキルなどの検索パラメーターを指定できる検索オプション。
* モバイルアプリでユーザーにポップアップとして表示される、管理者対応のカスタムアナウンス。
* アプリのスキルページでスキルを追加することはできません。
* オフライン時のコース受講。

詳しくは、「 [AdobeのLearning Managerモバイルアプリ](/help/migrated/learners/feature-summary/ipad-android-tablet-users.md).

## QRコードスキャンアプリを使用して、Learning ManagerのQRコードをスキャンします

Adobe版Learning Managerでは、QRコードをスキャンするアプリをダウンロードしなくても、ネイティブカメラアプリを使用して直観的にQRコードをスキャンできるようになりました。

ALMアプリのメニューを使用してQRコードをスキャンできる以前のワークフローをサポートするために、このリリースでは、新しいワークフローに切り替える方法に関する情報を提供しました。

QRコードベースのワークフローの機能強化により、このリリースより前に生成された古いQRコードはサポートされなくなります。 そのため、このリリース以降に行われる教室またはVCセッションで、以前のバージョンのLearning Managerを使用してQRコードを生成した場合は、新しいQRコードを生成する必要があります。

## 観察チェックリストの改善

作成者は、チェックリストのレビュー担当者として、マネージャーと店舗/場所マネージャーを選択できるようになりました。 マネージャーとストア/ロケーションマネージャーは、レビュー担当者として選択された場合、インストラクターの役割に切り替えることなく、チームのチェックリストを表示および完了することもできます。 インストラクターは引き続きチェックリストを確認します。 登録時にチェックリスト用に、インスタンスのレビュー担当者（インストラクター/マネージャー）に新しい「レビューチェックリスト」通知が送信されます。

マネージャーがチェックリストモジュールにレビュー担当者として追加されると、マネージャーアプリでチェックリストをレビューできるようになります。 インストラクターは引き続き、チェックリストを確認する必要があります。

詳しくは、「 [検証チェックリスト](/help/migrated/authors/feature-summary/courses.md#observation-checklist).

## その他の機能強化

### 学習者の検索

検索結果も次のように分類されます。

* 自分が最近行った検索
* 組織内でよく使われる検索

学習者の検索機能も改善されました。 学習者は、二重引用符「。..」、プラス「+」およびマイナス「 – 」演算子を使用して、関連する結果をより迅速に検索し、Google検索のようなエクスペリエンスを実現できるようになりました。

* 二重引用符(「。..」)を使用して、完全に一致するフレーズまたは単語を含むコースを検索します。 例えば、「data science」と入力すると、「data science」という語句で始まるコースが返されます。
* +演算子を使用して、特定の語句または単語を含む結果のみが表示されるようにします。 例えば、Computer Programming +pythonでは、「python」という単語を含むコンピュータープログラミングコースのみが表示されます。
*  – 演算子を使用すると、特定の語句や単語を含まない結果のみが表示されます。 例えば、 Computer Programming -pythonを指定すると、「python」という単語を含むコースを除くすべてのコンピュータープログラミングコースが表示されます。

### 学習目標の競合するポップアップ

学習者のセッションに競合がある場合、登録の際に概要ページにポップアップが表示されます。 カタログページまたは概要ページから登録する際に、学習者のセッションが既に競合している場合は、セッションの詳細が競合する競合ポップアップがトリガーされます。 学習者がセッションに競合がある場合でも、新しいセッションに登録できます。

![警告](assets/learner-pop-up.png)

このメッセージは単なる警告です。 コースには引き続き登録できます。

### 新しいテンプレート

「Session cancelled due to unenrollment」という名前の新しい電子メールテンプレートを追加しました。 ユーザーがトレーニングの登録を解除されると、ユーザーはキャンセル電子メールを受け取ります。

### コースの新しい価格設定機能

管理者は、管理者設定/基本情報セクションでアカウントの通貨を設定できるようになりました。 管理者は、記号とISO通貨コードの両方を指定できます（USD、GBPなど）。 すべての新しいアカウントのデフォルトは$です。 この変更は、学習者以外のアプリにのみ適用されます。 管理者は、学習者としてコースをプレビューするときに変更を表示できます。 通貨記号は、学習者のトランスクリプトとトレーニングレポートにも表示されます。

詳しくは、「 [管理者設定](/help/migrated/administrators/feature-summary/settings.md).

### インスタンスを切り替え

コースの特定のインスタンスに登録されている学習者は、コースで利用可能なすべてのインスタンスのリストを表示し、自分に適した別のインスタンスに切り替えることができます。 切り替えの理由は、学習者が前のインスタンスへの出席を取り消した場合や、新しいインスタンスのセッションタイミングが適している場合、または他の場合が考えられます。

学習者がコースの一部として達成した進捗状況、学習者のクイズスコアなどは、新しいインスタンスに引き継がれます。 この機能は、主に教室コースとミックス済みコースを対象としていますが、セルフペースのコースも含め、あらゆる種類のコースをサポートする予定です。

「すべてのインスタンスを表示」オプションが、「インスタンススイッチ」の設定に関係なく表示されます。 コースに複数のインスタンスがある場合、学習者はすべてのインスタンスを表示するオプションを表示できるようになりました。

「インスタンスの切り替え」オプションが有効になっている場合、学習者はコースを完了するまで様々なインスタンスに切り替えることができます。 このオプションが無効な場合、学習者はインスタンスを表示できますが、コースに登録することはできません。

学習者が以下を選択した場合 **すべてのインスタンスを表示**&#x200B;学習者は、すべてのコースインスタンスを確認できます。

![すべてのインスタンスを表示](assets/view-all-instances-new.png)

管理者は、学習者用のインスタンスを切り替えることができます。

![インスタンスを切り替え](assets/switch-instances.png)

![インスタンスを切り替え](assets/switch-instances-select.png)

学習者または管理者アプリからコースの任意のインスタンスが完了するまで、インスタンスを切り替えることはできません。

コースの作成中に、作成者は「インスタンスの切り替え」の有効と無効を切り替えることができます。 インスタンスの切り替えは、無料コースでのみ利用できます。

詳しくは、「 [インスタンス構成](/help/migrated/authors/feature-summary/courses.md).

**学習者**

学習者は、キャンセル待ちのコースインスタンスから別のインスタンスに切り替えることができます。 学習パスまたは資格認定のコースインスタンスを切り替えることもできます。

学習パスまたは資格認定に登録した学生は、コースページで「インスタンスを表示」をクリックしてインスタンスを切り替えることができます。

>[!NOTE]
>
>インスタンススイッチの構成は、マネージャーによりノミネートされた登録の種類ではサポートされていません。
>
>Flex LPからインスタンスを切り替える場合は、進行状況がもう一方のインスタンスに継承されます。

### 管理者によるアナウンスを通じて、学習者を対象としたアドホックメールを送信

管理者の方には、新しいタイプのお知らせ – 「電子メールとして」が追加されました。 この機能を使用すると、選択したユーザーグループの学習者や特定のトレーニングに登録している学習者に、対象となるアドホックメールを送信できます。

カスタム管理者と作成者も、必要なアクセス権がある場合は、このオプションをインスタンスレベルで表示できます。

詳しくは、「 [アナウンス](/help/migrated/administrators/feature-summary/announcements.md#as-email).

### 新しい電子メールテンプレート

このリリースでは、スキップレベルマネージャー用の電子メールテンプレートが追加されました。 これらのテンプレートは、マネージャーが、コース、学習パス、または資格認定の期限が欠落している第2レベルのレポートに関する電子メールを受け取った場合に使用されます。 また、第2レベルのレポートがこれらのコース、学習パス、または資格認定の完了期限に近づくと、受講者に電子メールが送信されます。

* コース完了のスキップレベルエスカレーション – 期限前
* コースの期日が過ぎている場合のスキップレベルエスカレーション
* 学習パス完了用のスキップレベルエスカレーション – 期限前
* 学習パスの期日が過ぎている場合のスキップレベルエスカレーション
* 資格認定の完了時にスキップレベルエスカレーションを行う – 期限前
* 資格認定期限に間に合わなかった場合のスキップレベルエスカレーション

このオプションはデフォルトで有効になっています。

### VCの教室

VCコースの場所ベースの検索を実行します。 新しい属性roomLocationがリソースモデルに存在し、VCコースの作成時に指定されたフリーフォームの場所を示します。

以下の変更が加えられました。

**LO**

learningObjects API用の新しいクエリパラメーターfilter.loFormat=Virtual Classroom。

**端点**: GET /primeapi/v2/learningObjects

**カレンダー**

カレンダAPIの新しいクエリパラメータfilter.allSessions=false。 パラメーターのデフォルト値はfalseです。 trueの場合、APIは学習者のすべてのカレンダーセッションを返します。

**端点**: `GET /primeapi/v2/users/<id>/calendar?filter.allSessions=false`

### 学習者の検索履歴

**検索**

新しいクエリパラメーターpersistSearchHistory。 デフォルト値はtrueで、検索候補のクエリが維持されます。

**端点**: GET /primeapi/v2/search?persistSearchHistory=true

**おすすめ**

新しいクエリパラメーターsuggestionType。 指定できる値は次のとおりです。

* learnerHistory （デフォルト）
* accountHistory

**端点**: GET /primeapi/v2/search/suggestions/?suggestionType=learnerHistory

### ユーザーグループのフィルタリング

学習目標および検索APIには、指定されたフィルターを介して、ユーザーグループに属するすべての学習目標を取得するためのフィルターが用意されています。 APIは、コンマ区切りのリストとしてフィルターをサポートしています。

新しいフィルターfilter.lo.announcementGroupsを提供し、提供されたユーザーグループフィルターに属するすべてのLOを取得できるようにしました。

これは、柔軟性を持たせるために、複数の値を持つコンマ区切りのリストにすることができ、処理は複数のグループ間の「OR」操作に基づきます。 つまり、指定したユーザーグループに該当するすべてのLOを取得します。
