# Adobe Learning ManagerとのSalesforce(SFDC)統合に関する問題のトラブルシューティング

## SFDC書き出しエラーのトラブルシューティング（2 ～ 3時間以上、書き出されない）

SFDCの書き出しが&#x200B;**2 ～ 3時間**&#x200B;以上正常に完了しない場合は、次の手順を実行して問題を特定し、理解してください。

1. **Salesforceホーム**&#x200B;ページに移動します。
2. **クイック検索**&#x200B;の検索ボックスに、**`Bulk Data Load Jobs`**&#x200B;と入力して開きます。
3. このページには、**過去7日間**&#x200B;の一括データ読み込みジョブが表示されます。
4. **Adobe Learning Manager (ALM)オブジェクト**&#x200B;に関連するジョブを検索します。
5. 調査する特定のジョブをクリックします。
6. ジョブの詳細ページの&#x200B;**下**&#x200B;までスクロールして、**エラーメッセージ**&#x200B;とその他の詳細を表示します。

多くの場合、根本的な原因は、**SFDCのフィールドレベルの権限が不十分です**。 失敗したフィールド名は次のようになります。

- `cp_Module_ID`
- またはALMカスタムオブジェクトに関連する他の`cp_...`フィールド。

このようなフィールドがエラーに表示された場合は、次のセクションに進みます。

## SFDCのALMカスタムオブジェクトフィールドにフィールドアクセス権を付与

**フィールドのアクセシビリティ**&#x200B;機能を使用して、特定のフィールド（例： `cp_Module_ID`）の権限の問題を修正します。

### 手順

1. **Salesforceホーム**&#x200B;ページに移動します。
2. **クイック検索**&#x200B;の検索ボックスに、**`Field Accessibility`**&#x200B;と入力して開きます。
3. オブジェクトの一覧から、関連する&#x200B;**レポートの種類/オブジェクト**&#x200B;を選択します。
   - 例：学習者トランスクリプト(LT)レポートの`cp_LearnerTranscript_xxxx_xxxx`。
4. 次のページで、**[フィールド別に表示]**&#x200B;を選択します。
5. **フィールドドロップダウン**&#x200B;で、書き出しエラーに表示されたフィールドを選択します
   - 例： `cp_Module_ID`。
6. アクセスマトリックスで、**システム管理者**&#x200B;プロファイル（および必要に応じてその他の関連するプロファイル）の&#x200B;**「非表示」**&#x200B;エントリをクリックします。
7. 次のページ：
   - 必要に応じて&#x200B;**「表示」**&#x200B;を有効にします。
   - ページ下部の「**保存**」をクリックします。
8. 保存後、フィールドアクセシビリティビューに戻ります。 **システム管理者** （および更新したその他のプロファイル）のフィールドは&#x200B;**&quot;編集可能&quot;**&#x200B;と表示されます。

## 影響を受けるすべてのフィールドに対してこの操作を繰り返し、書き出しを再試行します

1. **セクション2の**&#x200B;各フィールド&#x200B;**に対するフィールドのアクセシビリティ手順を**&#x200B;繰り返すと、**一括データ読み込みジョブ**&#x200B;でアクセス許可の問題が発生していると報告されました。
2. 問題のあるフィールドがすべて適切に表示され、編集権限が付与されたら、次の操作を実行します。
   - **Adobe Learning Manager**&#x200B;に戻ります。
   - **SFDCコネクタ**&#x200B;構成で、**エクスポートを再試行**&#x200B;してください。


## 重要な注意事項と制限事項

統合の設計またはトラブルシューティングを行う場合は、これらのSFDC-ALMの仕様を念頭に置いてください。

### SFDCにオブジェクト/フィールドの自動作成がない

- **SFDCコネクタでは、Salesforce**&#x200B;に新しいオブジェクトまたはフィールドは作成されません。
- **新しいフィールドがALM**&#x200B;に追加され、それをSFDCに表示する場合：
   - SFDCで対応するカスタムフィールド&#x200B;**を手動で**&#x200B;作成します。
   - SFDCカスタムフィールドをコネクタ構成の&#x200B;**適切なALMフィールド**&#x200B;に&#x200B;**マップ**&#x200B;します。
   - 新しいフィールドに&#x200B;**適切なフィールドレベルの権限**&#x200B;があることを確認してください（セクション2を使用）。

### カスタムドメインを持つALMアカウントのコールバックURL

- ALMアカウントで&#x200B;**カスタムドメイン**&#x200B;を使用している場合、エンジニアリングチームは、**そのカスタムドメイン**&#x200B;をSFDCコネクタのOAuth本番アプリの&#x200B;**コールバックURL allowlist**&#x200B;に追加する必要があります。
- これは、エンジニアリングへの&#x200B;**Jiraリクエスト**&#x200B;によって行われます。
- これがないと、コネクタのOAuthフローが失敗することがあります。

### タイムゾーンの制限（UTCのみ）

- SFDC組織が&#x200B;**UTC**&#x200B;以外のタイムゾーンを使用している場合、同期中に&#x200B;**完了および登録データが失われる**&#x200B;可能性があります。
- 理由： ALM **は現在、SFDC統合のUTCタイムゾーン**&#x200B;のみをサポートしています。
- 内部参照：追加のタイムゾーンをサポートするために、PAPI-24525が引き上げられます。

### データ型の制限のフィルター（選択リストとチェックリスト）

- ALMは、選択リストフィールド&#x200B;**で作成された** SFDCフィルターの読み込みをサポートしています。
- ALM **は、チェックリスト/複数選択チェックボックスフィールドに基づく**&#x200B;フィルターをサポートしていません。
- 理由： ALM **では、SFDCフィルターの文字列データ型**&#x200B;のみがサポートされます。
