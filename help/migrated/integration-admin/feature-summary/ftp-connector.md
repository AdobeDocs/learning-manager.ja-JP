# Adobe Learning ManagerのFTPコネクタ

## 概要

FTP（ファイル転送プロトコル）は、インターネットまたはローカルネットワークを介してクライアントとサーバー間でファイルを転送するために使用される標準のネットワークプロトコルです。 これにより、ユーザーはリモートサーバー上のファイルをアップロード、ダウンロード、および管理できます。 安全なファイル転送には、SFTP(SSH File Transfer Protocol)やFTPS(FTP Secure)などのバリアントが一般的に使用されます。 FTPは、Adobe Learning Managerと外部プラットフォーム間のユーザーデータやトレーニングデータの同期など、システム間のデータ交換を自動化するためにエンタープライズ環境で広く採用されています。

このドキュメントでは、Adobe Learning ManagerでFTPコネクタを設定および使用する方法を統合管理者に示します。 FTPコネクターを使用すると、安全なファイル転送プロトコルを使用して、Learning Managerと外部システムの間で自動データ交換を行うことができます。

FTP接続の設定方法、データフィールドのマッピング方法、ユーザーの自動インポートまたはエクスポートのスケジュール方法、同期アクティビティの監視方法について説明します。 このガイドは、外部学習プラットフォームまたは人事システムとのスムーズで安全な統合をサポートします。 社内ユーザーとxAPIステートメントを読み込み、ユーザースキル、学習者のトランスクリプトおよびxAPIデータを書き出すことができます。

統合管理者は、ユーザー、ユーザーデータ、学習コンテンツを移行するためのCSVファイルを生成し、Adobe Learning Manager FTPアカウントの指定されたフォルダーにアップロードする必要があります。 Adobe Learning Managerは、定義されたスケジュールに基づいてデータを読み取り、結合、および読み込みます。

これらの操作は、オンデマンドで実行するか、組織のニーズに合ったスケジュールを設定して実行します。

## FTP統合の利点

- データ管理における手作業と人為的エラーを削減します。
- 複数の外部ソースからのデータを同時に統合します。
- オンデマンドおよびスケジュールされたデータ操作をサポートします。
- 異なるシステム形式間で詳細なフィールドマッピングができます。

## 前提条件

FTPコネクタを設定する前に、環境が次の要件を満たしていることを確認してください。

- 統合管理者の役割とFTPコネクタの権限。
- ファイル転送に十分な帯域幅を持つ安定したインターネット接続。
- 必要なポートでFTPトラフィックを許可するファイアウォール構成。
- セキュリティ要件に応じて必要なポートアクセス

### 権限とアクセス

次の項目を確認してください。

- SSH鍵を生成および管理するためのアクセス（SSH認証を使用している場合）
- 指定したFTPフォルダーでCSVファイルを作成および更新する権限。

## 主な機能

### FTPコネクタによるデータの読み込みと書き出し

Adobe Learning ManagerのFTPコネクターを使用すると、外部システムとAdobe Learning Managerアカウント間のデータ交換が容易になります。 スケジュール設定されたオンデマンドの読み込みまたは書き出し操作をサポートし、手作業を減らし、正確で最新の情報を確保します。

この方法は、複数の外部システムとの統合をサポートします。 異なるシステムで個別のCSVファイルが生成される場合、Adobe Learning Managerはデータを結合し、1つのバッチとして読み込みます。

### Adobe Learning Managerへのデータの読み込み

_ユーザーデータのインポート_

構造化されたCSVファイルを指定されたFTPフォルダーにアップロードして、内部ユーザーデータを読み込みます。 Adobe Learning Managerは、設定されたスケジュールに基づいてこれらのファイルを読み取って処理し、ユーザー情報を最新の状態に保ちます。

_マルチソース統合_

複数の外部システムを使用している場合、各システムで独自のCSVファイルを生成できます。 Adobe Learning Managerでは、ファイルが結合され、データが1つのバッチとして処理されるため、様々なソースからのユーザーレコードを簡単に管理できます。

_xAPIのインポート_

コネクターはxAPI(Experience API)ステートメントもサポートしています。 サードパーティの学習システムからこれらを読み込み、複数のプラットフォームにわたる学習アクティビティを追跡してレポートできます。

### Adobe Learning Managerからのデータの書き出し

_学習者データの書き出し_

スキルの進捗状況、コースの完了、パフォーマンス指標などのユーザーデータを、指定されたFTPの場所に書き出します。 このデータは、外部レポートまたは分析に使用します。

_学習者のトランスクリプト_

コースの完了、資格認定、学習パスを含む詳細なトランスクリプトを生成して書き出し、コンプライアンスと資格情報検証をサポートします。

### 属性のマッピング

CSVファイルの列をAdobe Learning Managerユーザー属性にマッピングします。 必要に応じてマッピング設定を再利用および更新できるため、データ要件の変更に容易に適応できます。

### スケジュールと自動化

日次、週次、カスタム間隔などの定期的な間隔で実行されるように、インポートおよびエクスポートタスクをスケジュールします。 これにより、手作業を必要とせずに、データの更新が一貫して行われます。

## FTPコネクターの設定

FTPコネクターを設定して、Adobe Learning Managerと外部システム間で安全なデータ同期を確立します。

FTPコネクタを設定するには：

1. 統合管理者としてログインします。
2. **Adobe Learning Manager FTP**&#x200B;を選択し、**Getting Started**&#x200B;を選択します。

   ![](assets/ftp-connector1.png)
   _[はじめに]ボタンが表示されているAdobe Learning Manager FTPコネクタインターフェイス_

3. FTPコネクタのセットアップウィザードを続行するには、[**次へ**]を選択します。

   ![](assets/ftp-connector2.png)
   _構成ページに、FTPコネクタの設定を続行するための[次へ]ボタンが表示されます_

### 認証を設定する

Adobe Learning Managerでは、3つの認証方法がサポートされています。各認証方法には、異なるセキュリティレベルと複雑な要件があります。

#### 基本認証

この方法では、FTPアクセスに従来のユーザ名とパスワードのクレデンシャルを使用します。 実装はシンプルですが、SSHベースの代替手段よりもセキュリティが低くなります。

1. **[パスワードを使って基本認証を作成する]**&#x200B;を選択します。
2. 表示されたフィールドにFTPのユーザー名とパスワードを入力します。 続行する前に、資格情報が正しく入力されていることを確認してください。

   ![](assets/ftp-connector3.png)
   _ユーザー名とパスワードのフィールドを含むFTP認証フォームで、基本認証オプションが選択されています_

#### 既存のSSHキー認証

この方法は、セキュア認証のために既にSSHキーペアを確立している場合に使用します。

1. **既存のSSHキーを使用して認証を作成する**&#x200B;を選択します。
2. 公開キーのコンテンツを指定されたテキストフィールドにコピーして貼り付けます。 公開鍵の形式が正しいことを確認します（通常、ssh-rsaまたはssh-ed25519で始まります）。

   ![](assets/ftp-connector4.png)
   _公開キー入力用のテキストフィールドを含むSSHキー認証インターフェイス_

#### 新しいSSHキーを生成する

このオプションを使用して、このFTP接続用に新しいSSHキーペアを作成します。

1. **新しいSSHキーを生成して認証を作成する**&#x200B;を選択します。
2. **[SSHキーの生成]**&#x200B;を選択して、新しいキーペアを作成します。 生成された秘密鍵を安全にダウンロードして保存します。 公開キーはFTP接続用に自動的に設定されます。

   ![](assets/ftp-connector5.png)
   _[SSHキーの生成]ボタンとその他の構成オプションを含むSSHキー生成画面_

## FileZillaを使用してFTPに接続

FileZillaは、FTP接続を管理するためのオプションのツールです。 このツールは、ファイルを手動でアップロードする必要がある場合、ディレクトリ構造を確認する必要がある場合、または自動化されたAdobe Learning Managerプロセス以外で接続の問題をトラブルシューティングする必要がある場合に使用できます。

### FileZillaのインストールとセットアップ

FileZillaは、ファイル転送操作のための使いやすいインターフェイスを提供する無料のオープンソースFTPクライアントです。

FTPをFileZillaに接続するには、次の手順を実行します。

1. [公式Webサイト](https://filezilla-project.org/)からFileZillaをダウンロードしてインストールします。
2. **FileZilla**&#x200B;を開きます。
3. **ファイル**&#x200B;を選択し、**サイトマネージャー**&#x200B;を選択します。
4. **新しいサイト**&#x200B;を選択します。
5. 次の情報を入力します。
   - **FTPドメイン：**&#x200B;接続するFTPサーバーのアドレス（例： ftp.example.com）。 ホストドメインは、Adobe Learning Managerの「FTPコネクター」ページで確認できます。
   - **ポート：**&#x200B;既定のFTPポートは21です。 ただし、Adobe Learning Managerは安全な接続のためにポート22を使用します。
   - **FTPユーザー名：** FTPサーバーにアクセスするために必要なログイン名です。
   - **FTPパスワード：** FTPユーザー名にリンクされたパスワードです。
6. **Connect**&#x200B;を選択します。
7. 接続したら、ローカル（左）パネルとリモート（右）パネルの間でドラッグ&amp;ドロップしてファイルを転送できます。

## Adobe Learning ManagerでのFTPコネクタの使用

### FTPコネクタを使用した社内ユーザーの読み込み

ユーザー読み込み機能を使用すると、人事システムやその他の外部ソースからAdobe Learning Managerに従業員データを自動的に同期できます。

### マップ属性

属性マッピングは、外部データとAdobe Learning Managerのサポートされているデータ構造との間に接続を作成し、データが正しいフィールドに配置されるようにします。 この手順は必須です。

属性をマッピングするには：

1. **FTPコネクタ**&#x200B;ページで&#x200B;**内部ユーザー**&#x200B;を選択します。
2. **列マッピング**&#x200B;を選択します。
3. **マップの属性**&#x200B;ページで、次の操作を行います。
   - **左側**&#x200B;は、Adobe Learning Managerの必須フィールドを示しています。
   - **右側**&#x200B;には、CSV列の名前が表示されます。 最初は、この側には空のドロップダウンが含まれています。
   - 「**CSVを選択**」を選択して、サンプルCSVファイルをアップロードします。 これにより、右側のドロップダウンにCSVからの列名が入力されます。 [この記事](https://experienceleague.adobe.com/en/docs/learning-manager/using/integration/migration-manual#csv)を参照してください。
   - 各Adobe Learning Managerフィールドを対応するCSV列にマッピングします。

   ![](assets/ftp-connector6.png)
   _左側にAdobe Learning Managerフィールドが表示され、右側にCSV列ドロップダウンが表示された属性マッピングインターフェイス_

4. 「**保存**」を選択して、マッピングを完了します。

保存後、構成されたアカウントは管理者アプリで&#x200B;**データソース**&#x200B;として表示されます。 その後、管理者は読み込みをスケジュールしたり、手動同期をトリガーしたりできます。

### xAPIステートメントの読み込み

xAPIステートメントの読み込みにより、外部学習データをAdobe Learning Managerに取り込むことで、詳細な学習アクティビティトラッキングが可能になります。

_ソースの構成_

xAPIソース設定により、外部学習システムとAdobe Learning Managerのアクティビティトラッキング間の接続が確立されます。

ソースを構成するには：

1. xAPI構成セクションに移動します。
2. 構成の一覧で&#x200B;**新しい構成の追加**&#x200B;を選択します。

   ![](assets/ftp-connector7.png)
   _[新しい構成の追加]ボタンと既存の構成リストが表示された構成管理ページ_

3. **名前**&#x200B;と&#x200B;**ソースファイル名**&#x200B;を入力してください：
   - **名前：**&#x200B;このxAPIソースの説明的な識別子（LMS統合、外部トレーニングシステムなど）。
   - **ソースファイル名：** FTPフォルダーにアップロードされる正確なファイル名（ファイル拡張子を含め、正確に一致する必要があります）。

   ![](assets/ftp-connector8.png)
   _名前フィールドとソースファイル名フィールドを示す構成フォーム_

4. **保存**&#x200B;を選択して、基本的な構成を作成します。

_フィルターの追加（オプション）_

フィルターを使用すると、特定の条件に基づいてxAPIステートメントを選択的に読み込むことができます。

ソースのフィルタを追加するには、次の手順に従います。

1. 左側のウィンドウで[**フィルター**]を選択します。
2. **[新しいフィルターの追加]**&#x200B;を選択します。

   ![](assets/ftp-connector9.png)
   _[新しいフィルターの追加]ボタンがあるフィルター構成ページ_

3. 次の設定を行います。
   - **名前：**&#x200B;フィルタルールの記述名です。
   - **条件：**&#x200B;比較演算子（等しい、含む、より大きい、など）。

   ![](assets/ftp-connector10.png)
   _名前フィールドと条件を示すフィルター作成ダイアログ_

4. **新しいフィルターを追加**&#x200B;を選択して、さらにフィルターを追加します。
5. 必要に応じて、**アクション**&#x200B;列で「**保存**」または「**削除**」を選択します。
6. フィルターを追加したら、**[保存]**&#x200B;を選択します。

_フィールドのマップ_

フィールドをマッピングするには、次の手順に従います。

1. 左側のウィンドウで「**マッピング**」を選択します。
2. **マッピング**&#x200B;ページの左側にJSONフィールドパスが表示され、右側にCSV列の名前が表示されます。

   ![](assets/ftp-connector11.png)
   _インポートソースのマッピングを追加します_

3. デフォルトでは、次の必須フィールドをマッピングします。
   - **actor.mbox:**これは、学習者（実行しているアクター）の電子メールアドレスを表します
アクション)。 アクティビティを実行したユーザーを一意に識別します。
   - **verb.id:**これは、学習者が実行する操作の識別子です。たとえば、次のようになります
completed、attempted、またはpassed。 学習者のアクションを指定します。
   - **object.id:**これは、学習者が操作した学習オブジェクトまたはアクティビティを示します。
コース、モジュール、学習パスなど。
4. **新しいマッピングを追加**&#x200B;を選択して、追加のフィールドをマップします。
5. 各フィールドで、適切な&#x200B;**データ型** （文字列、数値、ブール値、または日付）を選択します。
6. 「**保存**」を選択して、マッピングを完了します。

## 読み込みのスケジュール

自動化されたスケジュール設定により、手作業を必要としない一貫したデータ同期が可能になります。
現在の学習活動レコードの保守

インポートをスケジュールするには、次の手順に従います。

1. 左側のウィンドウで[**スケジュールの構成**]を選択します。

   ![](assets/ftp-connector12.png)
   _有効なオプションとタイミング制御を示すスケジュール構成ページ_

2. **この接続を使用してxAPIステートメントのインポートを有効にする**&#x200B;を選択してください。
3. 自動インポートを設定するには、**[スケジュールの有効化]**&#x200B;を選択します。
4. 次のパラメーターを設定します。
   - **開始日：**&#x200B;スケジュールされたインポートをいつ開始するか。
   - **時刻：**&#x200B;インポートを実行する時刻です。
   - **後に繰り返す：**&#x200B;インポートを実行する頻度（毎日、毎週、カスタム間隔）。
5. 「**保存**」を選択します。

## オンデマンドで実行（オプション）

オンデマンド実行では、通常のスケジュールされた操作の外に即座にデータをインポートできます。

オンデマンドインポートを使用する場合：

- スケジュールを設定する前に新しい構成をテストする
- 緊急または時間に関係するデータ更新の処理
- 1回限りのデータ移行または修正を処理します。 読み込みの問題のトラブルシューティング

xAPIステートメントを手動で読み込むには：

1. 左側のウィンドウで[**オンデマンド**]を選択します。
2. 「**実行**」を選択します。

   ![](assets/ftp-connector13.png)
   _[実行]ボタンのあるオンデマンド実行ページ_

## 実行ステータスの表示

ステータス監視により、インポート操作をプロアクティブに管理し、問題を迅速に特定できます。

実行ステータスを表示するには、次の手順に従います。

1. すべてのインポート実行の一覧を表示するには、**実行ステータス**&#x200B;を選択します。
2. このページには、次の情報が表示されます。

   - **開始日：**&#x200B;インポート操作が開始された日時。
   - **期間：**&#x200B;処理に必要な合計時間。
   - **インポートの種類：**&#x200B;インポートがスケジュールされているか、オンデマンドであるか。
   - **現在の状態：**&#x200B;リアルタイムの状態情報。
      - **処理中：**&#x200B;現在実行中のインポート
      - **完了：**&#x200B;レコードカウントを使用した正常な完了
      - **失敗：**&#x200B;診断情報でエラーが発生しました

## 読み込みエラーのトラブルシューティング

実行ステータスセクションには、すべてのインポートタスクの包括的な概要が時系列で表示されるため、管理者は操作を監視して問題をすばやく特定することができます。

ステータスインジケーター：

- **成功：**&#x200B;インポートがエラーなしで完了しました。
- **警告記号：**&#x200B;実行中のエラーまたは問題を示します。
- **進行中：**&#x200B;現在実行中のインポート操作です。
- **保留中：**&#x200B;インポートがスケジュールされましたが、まだ開始されていません。

エラーが発生すると、インポートに失敗した場合の横に警告インジケータが表示されます。 エラーレポートリンクを選択して、詳細なエラーレポートをダウンロードします。
