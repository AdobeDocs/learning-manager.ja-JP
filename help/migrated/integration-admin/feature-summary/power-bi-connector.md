# Adobe Learning ManagerのPower BIコネクタ

## 概要

Power BIコネクタを使用すると、Adobe Learning ManagerとMicrosoft Power BI（商用ライセンス）を連携させて、学習データを分析、視覚化、共有できます。

この統合により、統合管理者は、学習者のトランスクリプト、ユーザースキル、xAPIアクティビティレポートなどのライブデータセットを、選択したPower BIワークスペースに直接自動的に書き出すことができます。

接続すると、Power BIのすべての機能を使用して、カスタムダッシュボードとレポートを作成できます。 これにより、組織は学習者の進行状況、スキルの達成、トレーニングの有効性に関する深い洞察を得て、リアルタイムの学習データに基づいて情報に基づいた決定を下すことができます。

>[!NOTE]
>
>Adobe Learning Managerは、Microsoft Power BIの商用版との統合のみをサポートしています。 Government Cloudバージョンとの統合はサポートされていません。

## 前提条件

- **商用ライセンス**&#x200B;のMicrosoft Power BIのみがサポートされています。
- Power BIアプリとワークスペースを作成する権限があることを確認します。
- **テナント名**、**アプリクライアントID**、**アプリクライアントシークレット**、**ワークスペースID**&#x200B;を取得します（オプション）。

## Power BIコネクタの設定

ALMとPower BIを接続するには、次の手順を実行します。

1. Adobe Learning Managerに統合管理者としてログインします。
2. **Power BI**&#x200B;コネクタタイルにカーソルを合わせ、**接続**&#x200B;を選択します。

   ![](assets/power-bi-connector1.png)
   _[接続]を選択してPower BIコネクタを構成する_

3. 次の情報を入力します。

   - クライアント ID
   - クライアントシークレット
   - テナント名
   - ワークスペースID （オプション）

   ![](assets/power-bi-connector2.png)
   _Power BIを構成するために必要な詳細情報を入力してください_

4. **Connect**&#x200B;を選択します。

## Power BIアプリの登録

Power BIアプリを登録するには：

1. [Power BIアプリの登録](https://app.powerbi.com/embedsetup)に移動します。
2. 「**組織の埋め込み**」を選択し、Microsoftアカウントにログインします。
3. アプリの名前を入力します。
4. **アプリの種類**&#x200B;の&#x200B;**サーバーサイドWebアプリ**&#x200B;を選択します。
5. [**リダイレクトURL**]セクションで、[**カスタムURLを使用**]を選択し、「[このURL](https://learningmanager.adobe.com/ctr/app/azure/_callback):」と入力します（環境で必要な場合は、ドメインを置き換えてください）。
6. [**ホームURL**]フィールドに、[このURL](https://learningmanager.adobe.com/)と入力します。
7. **アクセス許可**&#x200B;セクションで、**すべてのデータセットの読み取り**&#x200B;および&#x200B;**すべてのデータセットの読み取りと書き込み**&#x200B;を選択します。
8. Power BI管理者に連絡して、**テナント名**&#x200B;を取得してください。
9. Workspace IDがない場合は、Power BIに作業領域を作成し（Power BIProが必要）、URLからIDをコピーします。
10. 後で使用するために、**[アプリの登録]**&#x200B;を選択し、**クライアントID**&#x200B;と&#x200B;**クライアントシークレット**&#x200B;を保存します。

>[!NOTE]
>
>後で接続の再認証が必要になった場合は、新しいPower BIアプリを作成し、正しいリダイレクトURLを使用してください。

## Power BIへのレポートの書き出し

接続を設定したら、次のレポートを書き出すことができます。

- **学習者のトランスクリプト**
- **ユーザースキル**
- **xAPIアクティビティレポート**
- **統合レポート** （複数のレポートの組み合わせ）

### 学習者のトランスクリプト

#### 書き出しのスケジュール

1. 左パネルで「**学習者のトランスクリプト**」を選択します。
2. 書き出しページで「**スケジュールを有効にする**」を選択します。
3. **開始日**&#x200B;と&#x200B;**時刻**&#x200B;を選択してください。
4. エクスポートを繰り返す頻度（毎日、毎週など）の&#x200B;**間隔**&#x200B;を定義します。

   ![](assets/power-bi-connector3.png)
   _学習者のトランスクリプトのスケジュール書き出しを有効にする_

5. 「**保存**」を選択します。

#### オンデマンドで書き出し

- **開始日**&#x200B;を指定し、オンデマンドエクスポートを実行することで、手動でレポートを生成できます。
- レポートには、指定した日付から現在までのデータが含まれます。

### ユーザースキル

#### 書き出しのスケジュール

1. 左パネルで「**ユーザースキル**」を選択します。
2. 書き出しページで「**スケジュールを有効にする**」を選択します。
3. **開始日**&#x200B;と&#x200B;**時刻**&#x200B;を選択してください。
4. エクスポートを繰り返す頻度（毎日、毎週など）の&#x200B;**間隔**&#x200B;を定義します。

   ![](assets/power-bi-connector4.png)
   _ユーザースキルレポートのスケジュールエクスポートを有効にする_

5. 「**保存**」を選択します。

#### オンデマンドで書き出し

- **開始日**&#x200B;を指定し、オンデマンドエクスポートを実行することで、手動でレポートを生成できます。
- レポートには、指定した日付から現在までのデータが含まれます。

### xAPIアクティビティレポートの管理

**xAPIステートメント**&#x200B;をPower BIにエクスポートすることもできます。

#### xAPIの書き出しの設定

1. **[xAPIアクティビティレポートのエクスポート]**&#x200B;を選択します。
2. 左側のウィンドウで&#x200B;**構成**&#x200B;を選択します。

   - CSV列に一致するようにJSONパスのフィールドに入力します。
   - **[追加]**&#x200B;を選択して、他のパスを含めます。
   - **Edit**&#x200B;を使用してフィールドを更新します。
3. 「**保存**」を選択します。

#### 書き出しのスケジュール

1. **[スケジュールの構成]**&#x200B;を選択します。
2. **この接続を使用してxAPIステートメントのエクスポートを有効にする**&#x200B;を選択します。
3. **開始日**、**時刻**、および&#x200B;**間隔**&#x200B;を設定します。
4. 「**保存**」を選択します。

#### オンデマンド書き出し

1. **オンデマンド**&#x200B;を選択します。
2. **開始日**&#x200B;を指定します。
3. 「**実行**」を選択します。

>[!NOTE]
>
>Learning Record Store(LRS)の一部のxAPIステートメントで、JSONパスが設定されていない場合、その値はPower BI内でN/Aと表示されます。

#### 実行ステータスの表示

- **実行ステータス**&#x200B;を使用して、開始時間、期間、ステータスなど、書き出しの履歴を表示します。
- 警告アイコンは、実行に失敗したことを示します。 リンクをクリックして、エラーレポートを.CSVとしてダウンロードします。

### 統合レポート

**統合レポート**&#x200B;は、次のデータを結合します：

- 学習者トランスクリプト
- ゲーミフィケーション
- フィードバックレポート
- ログイン/アクセス
- ユーザースキル
- ユーザーレポート
- トレーニングレポート

これにより、Power BIをデータに結合して、より強力なダッシュボードを作成できます。

#### 統合レポート構成の作成

1. **統合レポート**&#x200B;を選択し、**構成**&#x200B;を選択します。
2. [**データセット名**]フィールドに一意の名前を入力します。
3. このデータセットに含める1つ以上のレポートを[**データエクスポートのレポートの選択**]で選択します。

   - 学習者トランスクリプト
   - ログイン/アクセス
   - トレーニングレポート
   - ゲーミフィケーション
   - ユーザースキル
   - フィードバックレポート
   - ユーザーレポート
4. [**ユーザーグループフィルターの追加**]フィールドを使用して、エクスポートするユーザーグループのデータを選択します。 既定では、**すべてのユーザー**&#x200B;が選択されています。
5. コンテンツカタログでレポートをフィルター処理するには、**[コンテンツカタログフィルターの追加]**&#x200B;フィールドを使用します。
6. フィルターテーブルは、**ユーザーグループ**、**カタログ**、または&#x200B;**時間**&#x200B;フィルターをサポートするレポートを表示します。

   ![](assets/power-bi-connector5.png)
   _統合レポートの構成を作成する_

7. レポートとフィルターを選択した後、右上の&#x200B;**保存**&#x200B;を選択します。

#### ステータス別に学習者のトランスクリプトをフィルタリング

- **すべて：**&#x200B;日付範囲内のすべてのレコード
- **完了：**&#x200B;完了した学習活動のみ
- **進行中：**&#x200B;進行中のアクティビティのみ
- **未開始：**&#x200B;未開始のレコードを除外します
- **未登録：**&#x200B;未登録のレコードを含みます

## Power BIテンプレートをダウンロード

Adobeには、すぐに使い始めるのに役立つすぐに使えるPower BIテンプレートが用意されています。

- テンプレートをダウンロードし、レポートを読み込んで、必要に応じてカスタマイズします。
- テンプレートを使用すると、ゼロから作業を開始することなく、魅力的なダッシュボードを作成できます。

## 学習パス関連の設定

レポートに&#x200B;**学習パス**&#x200B;がどのように表示されるかは、管理者の設定によって異なります。

- **既存の接続：**

   - **学習パス**&#x200B;が無効な場合、関連する行や列は含まれません。
   - 有効にすると、登録済み学習者の学習パス（上位レベル）がレポートに含まれます。

- **新しい接続：**

   - 学習パスが無効な場合、列には次が表示されます。

      - **埋め込みパス：**&#x200B;学習プログラム名。
      - 学習プログラムの&#x200B;**埋め込みパスID:** ID。
      - 学習パスに埋め込まれたコースの&#x200B;**埋め込みコースのID :**
   - 有効にすると、**型**&#x200B;列で関連する学習パス（上位レベル）が使用されます。
   - 新しい接続の場合、変更は30日後に適用されます。

### データの保存場所**

書き出されたすべてのPower BIは、データアカウントにデータセットとして表示されます。 これらを使用して、カスタムダッシュボードとビジュアライゼーションを構築します。
