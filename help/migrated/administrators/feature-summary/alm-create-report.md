# Report Builderテンプレートの概要

テンプレートは、Adobe Learning Managerによってすぐに使用できるレポート設定です。 各テンプレートは、登録と完了の追跡、コンプライアンスレポート、インストラクターのパフォーマンスなど、特定のユースケースに合わせて設計されています。 テンプレートを直接ダウンロードするか、テンプレートを複製して編集可能なコピーを作成できます。

1. Adobe Learning Managerに管理者としてログインします。
2. 左側のウィンドウで&#x200B;**レポート**&#x200B;を選択し、**Report Builder**&#x200B;を選択します。
3. 「**テンプレート**」タブを選択します。
4. 使用可能なテンプレートを参照します。各テンプレートには、ユースケースに応じた名前が付けられています。
   ![](assets/image007.png)
5. テンプレート名を選択して、読み取り専用プレビューを開きます。 この例では、「カタログ単位 – コース数MoM」テンプレートの近くで「複製」を選択します。 列、適用されたフィルター、および並べ替え順序を確認します。 テンプレートを複製すると、編集可能なコピーが開き、Report Builderの既存の設定が事前に読み込まれます。 レポートの名前、説明、列、フィルター、並べ替えはすべて編集してから保存できます。

## レポートの名前と説明

1. 「**名前**」フィールドで、デフォルトの名前（例えば、_カタログのコピー_ - _コース数MoM_）をレポートの一意の名前に置き換えます。 名前が必要です。
2. [**説明**]フィールドに、レポートに含まれる内容の簡単な概要を入力します。 これにより、他の管理者がレポートを表示または編集するときに、レポートの目的を理解しやすくなります。

## 列の追加と設定

「**列**」セクションには2つのパネルがあります。左側の「**列を選択**」と右側の「**選択した列**」です。

### 列を追加する

1. **列の選択**&#x200B;パネルで、名前を選択してデータセットを展開します。 例： **カタログ**&#x200B;または&#x200B;**アクティブフィールドユーザーグループ**。
2. 追加する列の横にある&#x200B;**+**&#x200B;アイコンを選択します。右側の&#x200B;**選択した列**パネルに列が表示されます。
   ![](assets/image009.png)
3. 同じ列を複数回追加します。 たとえば、同じフィールドに2つの異なる集計を適用する場合です。 その列に対して&#x200B;**+**&#x200B;をもう一度選択します。

### 列を並べ替え

**選択した列**&#x200B;パネルの任意の列の行の左側にあるハンドルをドラッグして、別の位置に移動します。 パネルの列の順序は、ダウンロードしたレポートの順序と一致します。

### 列の名前の変更

1. 列の行で&#x200B;**編集** （鉛筆）アイコンを選択します。
   ![](assets/image011.png)
2. エイリアスを入力します。エイリアスは、ダウンロードされたレポートの列ヘッダーとして、デフォルトのフィールド名の代わりに表示されます。
   ![](assets/image013.png)

### 列の削除

レポートから削除するには、列の行で&#x200B;**×**&#x200B;アイコンを選択します。

## グループ化の適用基準

**グループ化**&#x200B;コントロールは、**選択された列**&#x200B;パネルの上部に表示されます。

1. **グループ化条件を選択：**を選択します。
   ![](assets/image015.png)
2. グループ化する列を選択します。 複数のファイルを選択できます。 スクリーンショットでは、レポートは&#x200B;_カタログ_&#x200B;と&#x200B;_作成月_&#x200B;でグループ化されています。
3. 選択した各Group By列は、Group Byコントロールの下にタグとして表示されます。 グループ化された列を削除するには、そのタグで&#x200B;**×**&#x200B;を選択してください。

>[!NOTE]
>
>group byが適用される場合、group-by列ではない列はすべて集計関数が適用されている必要があります。 集約を含まない列では、エラーが発生します。

## 列への集約の適用

1. **選択した列**&#x200B;パネルの任意の非グループ化列で、**集計方法**&#x200B;を選択します。
2. ドロップダウンから関数を選択します。 スクリーンショットで、**学習目標** - **学習目標ID**&#x200B;は、**Count Distinct**&#x200B;を使用し、ount_of_courseとしてエイリアスされています。

使用可能な集計関数：

| 機能 | 戻り値 |
|----------|-----------------|
| カウント | グループ内の合計行数 |
| 固有値をカウント | グループ内の一意の値の数 |
| 条件付きカウント | 指定した値に一致する行の数 |
| 合計 | グループ全体の数値フィールドの合計 |
| 分 | グループ内の最小値 |
| 最大 | グループ内の最大値 |
| 平均 | グループ全体の平均値 |

## フィルターを適用

**フィルター**&#x200B;セクションは、**列**&#x200B;セクションの下にあります。 フィルタは、レポートに表示される行を制限します。

1. フィルターを追加するには、「フィルター」セクションの右側にある&#x200B;**+**&#x200B;アイコンを選択します。
2. フィルターを適用するフィールドを選択します。
   ![](assets/image017.png)
3. 演算子を選択し、値を入力または選択します。

既存のフィルターを編集するには、フィルター行の&#x200B;**鉛筆**&#x200B;アイコンを選択します。 ネストされたフィルタグループを追加するには、フィルタ行の右側にある括弧で囲まれた+アイコンを選択します。

## 並べ替えの設定

「並べ替え」セクションは、「フィルター」セクションの下にあります。

1. 並べ替えを追加するには、**+ [並べ替えの追加]**&#x200B;を選択します。
2. 並べ替えの基準にする列を選択し、**昇順**&#x200B;または&#x200B;**降順**を選択します。
   ![](assets/image020.png)
3. 繰り返して、2番目の並べ替えを追加します。 優先順位を変更するには、ソートする行の左側にあるハンドルをドラッグします。

>[!TIP]
>
>常に1つ以上の並べ替えを適用します。 並べ替えを行わないと、同じレポートのダウンロード間で行の順序が異なる場合があります。

## レポートを保存する

右上隅の&#x200B;**[レポートの保存]**&#x200B;を選択します。 レポートは&#x200B;**[レポート]**&#x200B;タブに保存され、ダウンロードする準備ができました。

## ベストプラクティス

* フィールド名の代わりに意味のあるヘッダー（_学習目標_ - _学習目標ID_&#x200B;など）をダウンロードできるように、すべての列にエイリアスを使用します。
* 一意のレコードが必要な場合は、「カウント」ではなく「個別のカウント」を使用します。たとえば、合計行ではなくカタログごとに個別のコースを使用できます。
* 特に共有または購読するレポートの場合、保存する前に並べ替えを適用します。
* 説明は最新の状態にします。 他の管理者は、レポートを開かずにレポートの範囲を理解するために、このレポートを使用します。
