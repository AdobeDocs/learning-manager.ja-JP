# Report Builderの傾向レポートを作成する

傾向レポートは、コース数、登録数、完了などの指標が時間の経過とともにどのように変化するかを示します。 日付列とトレンド細分性（日、週、月）を選択すると、Report Builderはその期間ごとにデータをグループ化します。

## トレンドデータの意味

Report Builderの傾向レポートには、**日付でグループ化された現在のデータのスナップショット**&#x200B;が反映されます。 各過去のデータの履歴状態は表示されません。

たとえば、月次登録トレンドは、作成された月に分散した現在の登録数を示します。 1月に登録した学習者の登録が解除されると、その登録レコードは表示されなくなります。 このレポートは、1月の実際のレコードではなく、現在のレコードの状態を反映しています。

これは、監査の目的では重要な区別です。 特定時点の履歴データが必要な場合は、このレポートを使用して、正確な履歴レコードではなく、方向性のある傾向を分析します。

## コース数トレンドレポートの作成

このレポートには、アカウントに追加されたコースの数が月ごとに表示されます。

1. **レポート** > **Report Builder**&#x200B;を選択し、[**レポート**]タブを選択します。
2. **[レポートの作成]**&#x200B;を選択します。 「コース数MoM」などの名前を入力します。
3. **学習目標ID**&#x200B;を&#x200B;**学習目標**&#x200B;データセットから追加します。
4. **作成日**&#x200B;を&#x200B;**学習目標**データセットから追加します。
   ![](assets/image039.png)
5. **作成日**&#x200B;に&#x200B;**グループ化**&#x200B;を適用します。傾向の粒度を&#x200B;**月**に設定します。
   ![](assets/image040.png)
6. **Count**&#x200B;を&#x200B;**学習目標ID**に適用します。別名「コース数」を入力します。
   ![](assets/image041.png)
7. **作成日**で昇順に並べ替えて、傾向を時系列に表示します。
   ![](assets/image042.png)
8. **レポートを保存**&#x200B;を選択し、**アクション** > **ダウンロード**&#x200B;を選択してレポートをダウンロードします。

ダウンロードしたファイルには、作成されたコースの数を示す、毎月のコース作成アクティビティのトレンドが含まれています。 コースの制作パターン、ピーク、減少、コンテンツ全体の増加を追跡するのに役立ちます。

## カタログ全体の完了トレンドレポートの作成

このレポートには、定義された期間のカタログごとの毎月の完了合計が表示されます。

1. **レポート** > **Report Builder**&#x200B;を選択し、[**レポート**]タブを選択します。
2. **[レポートの作成]**&#x200B;を選択します。 「カタログ補完MoM」などの名前を入力します。
3. **カタログ名**&#x200B;を&#x200B;**カタログ**&#x200B;データセットから追加します。
4. **モジュールのトランスクリプト**&#x200B;のデータセットから&#x200B;**完了日**&#x200B;を追加します。
5. **学習目標ID**&#x200B;を&#x200B;**学習目標**&#x200B;データセットから追加して、完了数をカウントします。
6. **カタログ名**&#x200B;に&#x200B;**グループ化**&#x200B;を適用します。また、**完了日**&#x200B;に&#x200B;**グループ化**&#x200B;を&#x200B;**か月**粒度で適用します。
   ![](assets/image_043.png)
7. **Count**&#x200B;を&#x200B;**学習目標ID**&#x200B;に適用します。 別名「完了数」を入力します。
8. フィルターを追加します： **カタログ**&#x200B;は、安全、POS、配信（またはアカウントに関連するカタログ）にあります。
9. フィルターの追加： **完了日**は昨年の期間内です。
   ![](assets/image044.png)
10. **完了日**で昇順に並べ替えます。
   ![](assets/image_045.png)
11. **レポートを保存**&#x200B;を選択し、**アクション** > **ダウンロード**&#x200B;を選択してレポートをダウンロードします。

## ベストプラクティス

* 完了の傾向には&#x200B;**完了日**&#x200B;を使用し、登録の傾向には&#x200B;**登録日**&#x200B;を使用します。 間違った日付フィールドを使用すると、誤解を招く結果が生じます。
* 日付フィルターを追加して、トレンドを具体的な期間に限定します。例えば、月間トレンドの場合は過去12か月、週間トレンドの場合は過去8週間などです。
* トレンド・レポートには、「Catalog completions MoM - last 3 months」のように、名前に詳細度と日付範囲を使用してラベルを付けます。これは、後で表示するときに明確になります。
