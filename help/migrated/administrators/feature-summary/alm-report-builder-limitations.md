# Adobe Learning ManagerのReport Builderの制限

Report Builderは、さまざまなカスタム・レポートのニーズに対応していますが、一部のレポート・タイプ、データ・セット、構成は、このリリースではサポートされていません。 このセクションでは、既知の制限事項と可能な場合の回避策について説明します。

## 繰り返し行われる資格認定

今回のリリースでは、繰り返し行われる資格認定に関する特定のクエリがReport Builderで正しく機能しません。 レポートに資格認定の再登録や再完了の追跡など、資格認定のサイクルが繰り返し含まれている場合は、代わりにAdobe Learning Managerの既存の資格認定レポートを使用します。

## サポートされていないデータセットおよびレポートタイプ

以下は、このリリースのReport Builderでは利用できません。

* **業務結果データ**: Report Builderでは、Bring Your Own Data (BYOD)統合はサポートされていません。 外部のビジネスデータとLMSデータを結合するレポートはサポートされていません。
* **監査レポート**:システム監査証跡および管理者アクティビティログは、Report Builderで構築できません。 このデータには、管理ビューの既存の監査レポートを使用します。
* **増分レポート**: Report Builderは、毎回フルスナップショットレポートを生成します。 前回のダウンロード以降に変更されたレコードのみを返す増分レポートはサポートされていません。 増分データの場合は、ジョブAPIを使用します。

## 傾向レポートの制限

### トレンドレポートごとに1つの日付列のみ

単一のトレンドレポートは、日付ベースの列でのみグループ化できます。 同じレポート内で2つの異なる日付ベースの傾向を組み合わせることはできません。

例えば、次のように構築できます。

* **月別の登録**&#x200B;を示すレポート（登録日別にグループ化）
* **完了を月別**&#x200B;に表示する別のレポート（完了日別にグループ化）

登録月と完了月の両方を別々のトレンド列として表示するレポートは1つしか作成できません。 両方を取得するには、2つのレポートを別々に作成し、それらを並べて分析します。

### トレンドには、履歴スナップショットではなく、現在のデータが反映される

Report Builderのトレンド・レポートは、履歴スナップショットではなく、現在のポイント・イン・タイム・データに基づきます。 レポートには、現在のレコードの状態が反映され、日付フィールド全体に分散されます。

これは、次のことを意味します。

* 1月に登録したが、その後に登録を解除した学習者は、1月の登録トレンドに表示されない場合があります
* 再編成されたユーザーグループは、過去のメンバーシップを反映しません
* 削除されたレコードは、アクティブだった期間のトレンドに表示されない場合があります

傾向分析を使用します。 正確な履歴レコードまたは監査の目的で、既存の固定レポートまたはジョブAPIの書き出しを使用します。
