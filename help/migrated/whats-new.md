# 新機能の概要 {#new-features-summary}

Adobe版Learning Managerの2024年3月リリースの新機能と機能強化について説明します。

## このリリースの最新情報 {#whatsnewandchanged}

### 外部ソースからのスキルの読み込み

それぞれのコネクタを使用して、LinkedInやGo1などのコンテンツプロバイダーからスキルを読み込みます。 この機能強化は、Learning Managerが外部のSkills CloudおよびTalent Managementシステムと統合できるようにするための目標の一部です。 読み込まれたスキルは、Learning Managerで管理者が定義したスキルに追加され、コースの作成ワークフローで作成者が利用できるようになります。 また、プラットフォーム全体のスキル検索機能が強化され、アカウントに多数のスキルが含まれている場合の検索機能が向上しました。

表示 [スキルの読み込み](administrators/feature-summary/import-skills-external-sources.md) もっと詳しく。

### カスタムブランディング

アカウントで使用可能なユーザーグループに基づいて、組織名、ロゴ、UIテーマなど、特定のUI要素をカスタマイズできるようになりました。 例えば、複数の部門がある組織では、各部門に表示するカスタムロゴとUIテーマを設定できます。

>[!NOTE]
>
>この複数のブランディング機能は、管理者のビューには適用されません。 アカウントに組織レベルのブランディングが常に表示されます。 これは、これは学習者向けの機能であり、管理者がアカウントに含めたくない場合があるためです。

表示 [複数のカスタムブランディング](administrators/feature-summary/themes.md#multiple-branding) 」を参照してください。


## 大規模なユーザーベースを持つアカウントの変更

### 管理者 – 「コース」または「学習パス」ページ

多数の学習者がコースに登録されている場合（50,000人を超える場合など）、学習者のリストは表示されません。 学習者は、 *学習者を検索* 検索バーを選択するか、 **ダウンロード** 検索バーの上部にあるリンクをクリックして、学習者のリストをダウンロードします。

### 管理者 – 学習者ページ

任意のユーザーを検索すると、 **学習者をダウンロード** および **エクスポート** オプションを選択すると、同じレポートがダウンロードされます。 一方、ユーザーグループを検索する際に、そのユーザーグループからフィルタリングされたユーザーをダウンロードできるようになりました。 ユーザーグループを検索する場合、 **学習者リストのダウンロード** 変更点 **ユーザーグループの学習者リストをダウンロード** この **エクスポート** オプションを選択すると、リスト全体が再びダウンロードされます。

### 管理者 – ユーザーページ

#### 社内ユーザー

ユーザー数が50,000人を超えた場合は、後で詳細な分析のためにデータをダウンロードするようにメッセージが表示されます。 検索バーが目立つようになり、形式にユーザーが表示されるようになりました *名前、電子メール | UUID*.

>[!NOTE]
>
>UUIDは、アカウントでUUIDが有効になっている場合にのみ表示されます。

#### 社外ユーザー

外部ユーザーの場合も、同じ動作が適用されます。 ユーザー数が多い場合は、ユーザーをダウンロードしたり、検索後にユーザーの詳細を取得したりすることもできます *名前、電子メール | UUID*.

#### ユーザークリーンアップページ

ユーザークリーンアップページで、削除されたユーザーに対して、 **削除日**. UUIDに基づいてのみ並べ替えることができます。

### 管理者 – インスタンスページ

#### コースまたは学習パス

登録数が多い場合、AdobeのLearning Managerには学習者の数が表示されません。 代わりに、学習者の数を選択し、学習者ページに移動できるアイコンが表示されます。

学習者の数は、おおよその値で表示されます。 例えば、学習者数が50,000人を超える場合、値は50K以上と表示されます。

### 管理者 – L1/L3ページ

L1フィードバックページで、コースの登録数が多い場合、学習者のリストが表示されません。 代わりに、後でより詳細な分析のためにユーザーのリストを書き出すことができます。

検索は先行入力をサポートし、結果は選択したインスタンスに制限されます。

#### 「出席とスコア付け」ページ

このページでは、ユーザーを検索すると、使用可能なすべてのインスタンスで検索が実行されます。 ただし、選択したインスタンスの結果は返されます。

「出席」ページでユーザーグループを検索したときに、ユーザーグループ内のユーザー数が登録に関係なく10,000人を超えた場合は、一括レベルのアクションのみを実行できます。 ユーザーのリストを表示できなくなります。

ユーザーグループのユーザー数が10,000人未満の場合は、個々のユーザーレベルのアクションを一括レベルのアクションとともに実行できます。 この場合、ユーザーのリストは無効になりません。

### 管理者 – 資格認定ページ

現在のバージョンのLearning Manager Adobeでは、資格認定に登録されているユーザーの数が多い場合、 **ステータス** ドロップダウンが無効になっています。

このリリースのLearning Manager Adobeでは、登録ユーザー数が多い場合、 **ステータス** ドロップダウンには2つのオプションのみが表示されます。 **登録済み** および **登録解除**. オプション **登録済み** はデフォルトで選択されています。 次を選択した場合 **登録解除**&#x200B;に登録されていない学習者のリストが表示されます。

#### ユーザーグループの変更

ユーザーグループの場合、ユーザーグループのユーザー数が50,000人を下回ると、 **ステータス** ドロップダウンに、「認定済み」、「割り当て済み」、「期限切れ間近」というオプションがすべて表示されます。

ユーザーグループのユーザー数が多い場合、 **ステータス** ドロップダウンには2つのオプションのみが表示されます。 **登録済み** および **登録解除**&#x200B;新しいデザインに従います。

## 高度な検索機能

このリリースでは、検索機能が強化されています。 検索結果は、メタデータだけでなく、意味的、コンテンツ内検索に基づいてフェッチされ、精度、最新性、および関連コンテンツに基づいて結果を導出する。

この変更は次の場合に反映されます。
* カタログと「学習状況」ページ:コース、学習パス、資格認定に対するホバーアクションが削除されました。
* 検索バーの外観。
* 学習アプリにフィルタータグを追加。

検索機能を有効にするには、Learning Manager AdobeのCSAMチームまでお問い合わせください。

## レポートの変更

* トレーニングレポートの「タグ」列と「スキル」列が「タグとスキル」に変更されました。
* レポートが追加されました [ゲーミフィケーション監査証跡](administrators/feature-summary/reports.md#gamification-audit-trail).
* アカウントにスキルに割り当てられた学習者が280000人以上ある場合、スキル学習者レポートがzip形式のcsvとしてダウンロードされます。
アカウントに学習者が250000人未満の場合は、同じレポートがCSVとしてダウンロードされます。
管理者ページで、次を選択します **管理者** > **スキル** > **スキル** > **学習者**. レポートがCSVとしてダウンロードされます。
* この [セッションの概要レポート](administrators/feature-summary/reports.md#session-summary-report) には、[ロケーション情報]と[ロケーションリージョン]という2つの新しい列があります。

## チェックリストの機能強化

2024年3月にリリースされたAdobe Learning Managerでは、以下のチェックリストワークフローが強化されています。

### チェックリストの失敗に関する進行状況を禁止する

作成者はチェックリストの作成時に、 **有効にする** （「必須チェックリスト」セクション）。 これにより、学習者がチェックリストに不合格になった場合に、モジュール内で学習を進めなくなります。 チェックリストに合格した場合にのみ続行できます。

その後、チェックリストのレビュー担当者（インストラクターやマネージャー）は、チェックリストのステータスを確認できます。 レビュー担当者は、順序が間違っている学習者のチェックリストをレビューすることもできます。

### チェックリストの再評価

作成者はチェックリストの作成時に、 **有効にする** をクリックします。 これにより、マネージャーやインストラクターは、チェックリストに合格するまで、学習者を再評価することができます。

モジュールが必須の場合、再評価チェックボックスはデフォルトで選択されています。

再評価が有効な場合、インストラクターやマネージャーはチェックリストのステータスを「失敗」から「合格」に変更することもできます。

「チェックリスト」ページでは、インストラクターは保留状態の学習者の数を確認できます。 インストラクターは、学習者を評価して、合格または不合格にすることができます。 学習者が失敗状態の場合、再評価が有効になっていないときにのみチェックリストを表示できます。

これは、 **有効にする** チェックリストの作成中に、「再評価」セクションでチェックボックスが選択されていませんでした。 このチェックボックスを選択すると、インストラクターのチェックリストページに「表示/再評価」ボタンが表示されます。

このボタンを選択すると、学習者を再評価して、合格または不合格をマークできます。

>[!NOTE]
>
>これらの機能は、再評価とチェックリストの作成の両方とも、新しく作成されたモジュールにのみ適用されます。 公開されたコースのオン/オフを切り替えることはできません。


表示 [チェックリストの作成](authors/feature-summary/courses.md#checklist-fail) 」を参照してください。

## その他の機能強化

### セッション関連の電子メール通知

以前のリリースのAdobeであるLearning Managerでは、学習者がセッション関連の電子メール、セッションの詳細の更新、セッションの招待、セッションのリマインダーで、次の状況を行っていませんでした。

* 学習者がコースを完了すると、
* コースに新しいセッションが追加された場合、または
* 既存のセッションに変更があります。

2024年3月にリリースされたAdobe Learning Managerでは、以下の点が新しく変更されました。

* セッションの詳細が更新され、セッションが招待されました（学習者およびインストラクター用）
   * 今後のセッションでは、次のメール **セッションの詳細が更新されました**, **セッションへの招待** 登録済み学習者および現在のインストラクターは非推奨となります。 過去のセッションの場合：電子メール **セッションの詳細が更新されました** および **セッションへの招待** 登録済み学習者と現在のインストラクターは、そのままの状態が維持されます。
* リマインダーメール（管理者および学習者向け）
   * 以降のセッションのみ **セッションのリマインダー** メールが送信されます。

>[!NOTE]
>
>メールは、セッションやコースの完了に依存しません。


### AEM参照サイトの変更

AEMの参照サイトで、学習者アクセストークンに管理者更新トークンを追加する機能がサポートされるようになりました。

### インストラクターからの提出物を非表示

学習者がファイル提出ワークフローを使用してファイルをアップロードした後、提出に対してインストラクターがアクション（承認または却下）を行わない場合、提出URLは事前に定義された日数が経過するとビューに表示されません。 日数を設定または変更するには、CSAM Adobe版Learning Managerにお問い合わせください。

### 製品用語の変更

### モバイルアプリの変更

このモバイルアプリリリースでは、学習者は期限切れのコースリマインダーをスケジュールおよび管理できます。 期限切れのリマインダー通知をクリックすると、次のオプションにアクセスできます。

* キャンセル
* コースを開く
* 3日後にもう一度通知する
* 1週間以内に再度通知する

Androidの場合：プッシュ通知をクリックすると、 **コースの概要** ページです。
iOSの場合：プッシュ通知をクリックすると、アプリのホームページが表示されます。 これは、iOSでは既知の制限です。

### Salesforceの学習者アプリのチェックリストの変更

学習者がチェックリストに失敗すると、次のモジュールまたはコースに進めません。 「必須チェックリスト」チェックボックスが選択されていると、学習者がチェックリストに失敗した場合、コースを先に進めません。

Webアプリの場合と同様に、Salesforceアプリのチェックリストに失敗した場合、学習者にはメッセージが表示され、先に進みません。

### Connect VCの変更点

AdobeのLearning Managerの現在のリリースでは、学習者に **未出席** connect VCセッションに登録されているが、完了条件を満たしていない場合。

このリリースでは、ステータスがに変わります **まだマークされていません**.

## このリリースでの API の変更

### 学習者API

このリリースでは、学習者がユーザーグループレベルでブランディングロゴとパーソナライズされたテーマを表示するためのAPIサポートが追加されました。

APIの/accountおよび/user?include=accountは、logoUrl、logoStyling、およびthemeDataに属するユーザーのアクティブフィールドに固有の4つのフィールドを返します。

### 新しい属性

learningObjectResourceの新しい属性isExpiredSubmissionは、リソースの送信が期限切れかどうかを示します。

* GET /account API：新しい属性を返します **expireSubmissionDuration** X。ここで、Xは設定された日数です。 設定されていない場合は、0が返されます
* GET /LO APIとresourceに新しい属性が含まれています **isExpiredSubmission**「正誤問題です。
   * 送信の有効期限が切れ、「submissionUrl」が表示されない場合は、Trueを指定します。
   * Falseの場合、送信の有効期限は切れず、「submissionUrl」が取得されます。

### チェックリストのAPIの変更

1つのコースは複数のモジュールで構成され、チェックリストはモジュールの一種です。 このチェックリストモジュールはインストラクターによって評価され、評価に基づいて「失敗」または「成功」とマークできます。

ただし、どちらの場合もチェックリストのステータスは「完了」とマークされ、この方法でコースは「完了」とマークされます。

このリリースのLO APIには次のパラメーターが含まれています *isChecklistMandatory*. 値がTrueの場合、チェックリストは必須です。

### 複数のロケールのサポート

管理者は、選択した言語でL1フィードバックレポートをダウンロードできるようになりました。 ただし、Power BIに関するL1フィードバックレポートはまだダウンロードできません。 APIリクエストで、パラメーターpreferredLocaleを使用して、選択したロケールを指定します。

### インスタンスの概要の数の変更

これは、教室/VCコースの登録が1,000人を超えるアカウントに適用されます。

この数が1000未満の場合、登録によってキャッシュが無効になり、更新後の値（登録数、完了数、seatLimitなど）がGET・サマリーAPI呼び出しで戻されます。

アカウントがこの機能に対して有効になっていて、登録数が1000を超える場合は、値がキャッシュから取得されます。

### 非推奨のパス

現在、Learning Manager APIはグラフデータ構造に従っているため、インクルードを介してAPIモデルを走査することでデータを取得できます。 最大7つのレベルのAPIをトラバースできますが、単一のAPI呼び出しを使用したデータの取得は計算上、高コストです。

既存および新規のお客様は、1回の大電話ではなく、何度も少額電話をかけることをお勧めします。 この方法では、不要なデータがコールにロードされるのを防ぐことができます。

#### 非推奨のパス

次のパスは非推奨です。

* /learningObjects
   * 非推奨のパス：
      * enrollment.loInstance.loResources.resources
      * instances.loResources.resources
   * 既存のパス：
      * enrollment.loInstance
      * instances.loResources
* /learningObjects/{id}
   * 非推奨のパス：
      * enrollment.instances.subLoInstances.learningObject
   * 既存のパス：
      * enrollment.instances.subLoInstances
* /enrollments
   * 非推奨のパス：
      * loInstance.learningObject.enrollment
   * 新しいパス：
      * loInstance.learningObject
* /learningObjects/{id}
   * 非推奨のパス：
      * instance.subLoInstances.learningObject.enrollment.loResourceGrades
   * 新しいパス：
      * instance.subLoInstances

### ジョブAPIのログインアクセスおよびユーザー監査レポートのアーカイブの変更

このリリースでは、ジョブAPIは、ログインアクセスレポートを最大5四半期、ユーザー監査レポートを6か月間保持します。 この期間よりも古いデータをダウンロードする場合は、アーカイブ・パラメータを渡し、四半期と年を指定する必要があります。 サンプルペイロードを参照してください。

```
{
    "data": {
        "type": "job",
        "attributes": {
            "description": "description of your choice",
            "jobType": "generateLoginAccessReport",
            "payload": {
                "fromDate": "2023-04-01T18:30:00.000Z",
                "toDate": "2023-04-30T18:30:00.000Z",
                "archive": {
                    "quarter": "4",
                    "year": "2021"
                }
            }
        }
    }
}
```

ダウンロードを試みた場合 **ログインアクセス** 四半期が5つ以上のレポートには、エラー・メッセージが表示されます。 ダウンロードしようとすると、同様のエラーメッセージが表示されます **ユーザー監査** 6ヶ月を超えるレポート。

## このアップデートで修正されたバグ {#bug-fixes}

* 学習者がコースに登録されている状態で、別のコースに登録しようとすると、警告メッセージが表示されます。
* ユーザーグループは、削除された後も「検索」に表示されます。
* ユーザーが大量のデータを含む多くの学習者トランスクリプトをトリガーすると、学習者トランスクリプトキューがブロックされ、新しいリクエストを防止できます。
* 子アカウントが親アカウントにレポートの共有を要求した場合、親アカウントは要求できません。
* コースおよび学習パスのURLが正しくない場所にリダイレクトされます。
* 学習者が、カタログページでコースリンクをクリックすると、別のコースのコースインスタンスが断続的に表示される。
* この **登録解除** 最初の登録後にボタンは正常に表示されませんが、更新後にボタンが表示されます。
* 名前に空白が含まれているコンテンツまたはクイズを保存できません。
* マネージャーが承認したコースでは、ユーザーグループに学習者を再登録できます。
* 場合によっては、追加のアクティブフィールドを追加しようとすると、「アクティブフィールドを保存できませんでした。」というエラーメッセージが表示されることがあります。
* 関連するコースセクションのコースカード内のコース名に、テキストがオーバーフローします。
* インスタンスを切り替えて学習者をインスタンスに登録した後も、古いインスタンスはOutlookカレンダーに残ります。
* ピアアカウントの学習者がコースのサムネールを選択しようとすると、エラーメッセージが表示されます。
* 学習者がコースに登録すると、登録に関する複数の通知が届きます。
* コネクタで作成されたカタログの名前をユーザーが手動で変更すると、新しいカタログが作成され、コースが正しくないカタログに公開されます。
* 非アクティブなアカウントに属しているユーザーは、引き続きサブスクリプションメールを受信します。

### API関連のバグ修正

* API GET/ユーザーは、マネージャーの詳細を取得しません。
* あるアカウントで、スケジュールされたダウンタイム中に、スケジュールされたFTPユーザーの読み込みによってユーザーが作成されました。
* モバイルアプリまたは没入型モードで、コースインスタンスを削除または廃止し、次のアクティブなインスタンスを選択した後、 **関心のある項目を登録** 代わりにボタンが表示 **登録**.
* ピアアカウントの学習者が学習目標APIを使用してコースのサムネールを選択しようとすると、エラー「403禁止」が表示されます。
