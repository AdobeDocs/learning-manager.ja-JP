# Adobe Learning Managerの管理アカウントタイプ

## ALM役割のマッピング

Adobe Learning Managerでは、最上位の管理アカウントが管理者ロールになります。 この役割は、このガイドでFedRAMP用語の「トップレベル管理アカウント」を使用する場合に常に参照されます。 カスタム管理者と統合管理者は、特権（スコープ指定）アカウントとして扱われます。

次の表は、FedRAMPアカウント層をAdobe Learning Managerで使用される具体的なロールにマッピングします。

| FedRAMP項 | ALMロール名 | 説明 |
|--------------------------------------|----------------------------|-------------|
| 最上位の管理アカウント | は、 | ALMで最も権限の高い役割。 ユーザー、役割、学習コンテンツ、レポート、統合、およびすべてのシステム構成を完全に制御できます。 FedRAMPの最上位レベルの管理アカウント定義に直接マップされます。 |
| 特権アカウント（スコープ） | カスタム管理者 | 特定の機能（レポート作成、カタログ管理など）を範囲とする管理者権限の定義済サブセットです。 完全なアカウントレベルの制御がない。 |
| 特権アカウント（統合） | 統合管理者 | ALMと外部システム間の統合とAPIベースのアクセスを管理します。 管理者特権は統合管理のみにスコープが設定されます。 |


Adobe Learning Managerでは、ロールベースのアクセス制御(RBAC)モデルを使用して、管理アクセスを管理します。 管理者の役割は、許可された管理者のみが割り当てます。

詳細については、[Adobe Learning Managerのカスタムロール](https://experienceleague.adobe.com/ja/docs/learning-manager/using/admin/custom-role)を参照してください

## IDタイプと推奨される認証

Adobe Admin Consoleでは、管理者アカウントに対して3つのidタイプをサポートしています。 IDタイプの選択には、直接のセキュリティ上の影響があります。

| IDタイプ | 説明 | セキュリティの推奨事項 |
|---------------------------|-----------------------------------------------------------------------------|------------------------------------------------------------------------------------------|
| Adobe ID（個人ID） | デフォルトのタイプ。Adobeによって管理されます。 誰でも作成できます。 | 管理者には推奨されません。 組織はこのアカウントタイプを制御できません。 |
| Enterprise ID | Admin Consoleシステム管理者によって管理される組織所有のアカウント。 | Federated ID/SSOが利用できない場合は問題ありません。 2FAを強制します。 |
| Federated ID(SSO) | 組織所有。SAML 2.0 SSOと統合されています。 組織は認証を完全に制御します。 | おすすめ。 組織のIdPを介して認証します。IDプロバイダーレベルでのMFA強制をサポートします。 |

詳しくは、以下を参照してください。

* [IDタイプ](https://helpx.adobe.com/jp/enterprise/using/admin-console.html)
* [安全なユーザー認証とパスワード](https://helpx.adobe.com/jp/enterprise/using/authentication-settings.html)

## 役割の割り当てとアクセス制御

Adobe Learning Managerの管理アカウントへのアクセスは、既存の管理者によって明示的な[ロールの割り当て](https://experienceleague.adobe.com/ja/docs/learning-manager/using/admin/user-management/add-users-user-groups)によって制御されています。 安全な管理アクセスの主な特徴は次のとおりです。

* 管理者の役割は、許可された管理者のみが割り当てます。
* アクセスは役割に基づいており、割り当てられた権限に応じて範囲が設定されます。
* 組織は、正当なビジネスニーズを持つユーザーへの管理アクセスを制限する責任があります。

Adobeでは、管理者アクセスを定期的に確認して、最低限の権限の原則に確実に従うことをお勧めします。

## 多要素認証(MFA)の適用

Adobeでは、組織全体で2段階認証(2FA)を実施することを強くお勧めします。 MFAは、Enterprise ID版およびAdobe IDをご利用のお客様に適用されます。 Federated IDユーザーは、組織のIDプロバイダーでMFAを実施する必要があります。

Adobe Admin Consoleで2FAを強制するには、次の手順を実行します。

1. Adobe Admin Consoleにログインします。
2. 設定/プライバシーとセキュリティ/認証設定に移動します。
3. 2段階認証を有効にして、「強制」オプションを選択し、ユーザーが無効にできないようにします。
4. オプションで、IPベースのアクセス制限と高度なセッション設定（「最大セッション寿命」/「最大アイドル時間」）を構成します。

>[!IMPORTANT]
>
>Adobeでは、2FAを施行し、オプションのままにしないことをお勧めします。 2FAの適用には最大24時間かかる場合があります。 Federated IDユーザーの場合は、IDプロバイダーでMFAを強制します。

詳細については、[セキュリティで保護されたユーザー認証](https://helpx.adobe.com/jp/enterprise/using/authentication-settings.html)を参照してください。


## 管理者としてログイン

ALM [管理者](https://experienceleague.adobe.com/ja/docs/learning-manager/using/get-started/getting-started-admin)は、Admin Consoleを通じて管理されている組織の資格情報を使用して、ALMプラットフォームに直接サインインします。

### 管理者ロールの割り当て

ALMプラットフォーム内で、管理者はロールの作成と管理、ユーザーへのアクセス権の割り当て、運用責任に基づくアクセス権範囲の定義を行うことによって、管理アクセス権を設定します。

ALMに管理者ロールを割り当てるには：

1. 既存のAdministratorとしてAdobe Learning Managerにログインします。
2. ユーザー/社内に移動します。
3. ターゲット・ユーザーを検索または選択します。
4. 「アクション」 > 「役割の割り当て」 > 「管理者にする」の順に選択します。

カスタムの管理ロールを使用すると、アカウントレベルの権限を一元的に制御しながら、管理タスクを委任できます。 カスタム管理者は、特定のユーザーグループまたはカタログに対してスコープを設定できます。

詳細については、[ユーザーとユーザーグループの追加](https://experienceleague.adobe.com/ja/docs/learning-manager/using/admin/user-management/add-users-user-groups)を参照してください。

## ログイン方法とSSOの構成

ALM管理者は、セキュリティに関連する重要な設定である、設定/ログイン方式を介して、すべてのユーザーが使用できるログイン方法を制御します。

* **社内ユーザー**:ログインモードをAdobe IDまたはシングルサインオン(SSO)に設定します。 SAML 2.0経由のSSOを強くお勧めします。
* **社外ユーザー**:ログインモードをAdobe ID、SSO、またはLearning Manager IDに設定します。 選択した方法を組織のセキュリティポリシーに合わせます。

Adobeでは、すべての社内ユーザーのログイン方法として、Federated ID/SAML 2.0 SSOを使用することをお勧めします。 これにより、組織のIDプロバイダーによって認証が完全に制御され、MFAの集中適用と、ユーザーの出発時にアカウントが即座に取り消されます。

詳細については、[設定](https://experienceleague.adobe.com/ja/docs/learning-manager/using/admin/settings)を参照してください。

## プロビジョニング時に推奨される安全なデフォルト値

ALMアカウントが最初にプロビジョニングされる際には、管理Adobeに操作可能なアクセス権を付与する前に、次のデフォルト設定を確認することをお勧めします。

| 設定 | 推奨される安全なデフォルト | 設定する場所 |
|------------------------------|------------------------------------------------------------------|-------------------------------------------------|
| ログイン方法 – 社内ユーザー | シングルサインオン(SSO)/Federated ID | ALM/設定/ログイン方式 |
| 2段階認証(2FA) | 強制（どのユーザーでもオプションではありません） | Admin Console/設定/プライバシーとセキュリティ |
| 最大セッション期間 | 8時間または組織ごとのポリシー | Admin Console/設定/詳細設定 |
| 最大アイドル時間 | 30分または組織ポリシーごと | Admin Console/設定/詳細設定 |
| 管理者ロールの範囲 | 最小権限。可能な場合は、カスタム管理者の役割を使用します。 | ALM/ユーザー/カスタムの役割 |
| 外部ユーザーの有効期限 | すべての社外ユーザープロファイルに有効期限を設定する | ALM/ユーザー/外部 |

### 新しい管理者向けの初期設定チェックリスト

新しい最上位の管理者アカウントをプロビジョニングする場合は、操作アクセス権を付与する前に次の操作を行います。

* IDタイプが個人Adobe IDではなく、Enterprise IDまたはFederated IDであることを確認します。
* Admin Consoleレベルで2FAを強制します([Settings] > [Authentication Settings])。
* まだ有効になっていない場合は、SSO/Federated IDを構成します。
* Admin Console/設定/詳細設定で、最大セッション期間と最大アイドル時間を設定します。
* 「管理者ロール範囲を制限」 – ユーザーの責任に必要な権限のみを割り当てます。
* 管理者アカウントが、ITチームによって管理されている組織の電子メールアドレスに関連付けられていることを確認します。
* 組織の特権付きアクセス権登録簿にアカウントを記録します。

## 管理口座の操作

### 日常的な管理タスク

管理アカウントは、次のような日常的な運用タスクを実行するために使用されます。

* **ユーザーライフサイクル管理**:ユーザーの作成、プロファイルの更新、および役割の変更を行っています。
* **学習コンテンツ管理**:コース、学習プログラム、資格認定、カタログを管理します。
* **レポートと分析**：学習者の進行状況とプラットフォームの使用状況に関するレポートを生成およびレビューします。
* **統合とシステム構成**:コネクタ、APIベースのアクセス、およびシステムレベルの設定を管理します。

管理者は、管理操作を実行する際に、組織の内部アクセス制御に従い、管理ポリシーを変更する必要があります。

[Adobe Learning Manager管理者のよくある質問](https://experienceleague.adobe.com/ja/docs/learning-manager/using/faq/frequently-asked-questions-for-administrators)を参照してください


### ロールの階層と委任

Adobe Admin Consoleでは、階層構造の管理構造を使用します。 システム管理者は、権限の低い役割に責任を委任して、トップレベルの管理者アカウントの攻撃を軽減することができます。

* 製品管理者：特定のAdobe製品(Adobe Learning Managerなど)へのアクセスを管理します。
* 製品プロファイル管理者：特定の製品プロファイルのユーザーメンバーシップを管理します。
* ユーザーグループ管理者：ユーザーグループのメンバーシップを管理します。
* ALMカスタム管理者：カタログおよびユーザーグループごとに設定可能な権限を持つ、ALM内のスコープ指定された管理者。

### 継続的なガバナンス

以下の原則に従い、組織はALM管理者アカウントを継続的に運用する必要があります。

* **定期的なアクセスレビュー**:Admin Console内のシステム管理者（ユーザー/管理者）およびALM管理者（ユーザー/内部）のリストを定期的に監査し、現在の権限のあるスタッフだけがこれらのロールを保持するようにします。
* **監査ログの監視**: Admin Console監査ログには、管理者が行ったすべての変更が記録されます。 システム管理者は、すべての情報を把握できます。 承認されていない変更がないか、定期的にログを確認してください。
* **最低限の常時アクセス**：日常的なタスクに最上位の管理者アカウントを使用しないでください。 特に必要なタスクについては、完全な管理者アクセス権を予約してください。
* **セッションセキュリティ**：無人セッションからの露出を制限するには、[Admin Console] > [設定] > [詳細設定]で[最大セッション期間]と[最大アイドル時間]を構成します。

詳細については、[Admin Consoleの概要](https://helpx.adobe.com/jp/enterprise/using/admin-console.html)を参照してください。

### 管理者コントロールによるユーザーアカウントの管理

ALM管理者は、内部および外部のユーザーアカウントを管理します。 セキュリティ関連の操作には、次のものがあります。

* ユーザーの自動削除：設定で、管理者は、非アクティブな内部ユーザーを指定日数が経過すると自動的に削除するように設定できます。これにより、休止中のアカウントリスクが軽減されます。
* 外部ユーザーの有効期限：管理者は、外部ユーザープロファイルを作成するときに有効期限を設定します。 期限切れのアカウントは、自動的に非アクティブ状態に移行します。
* ユーザーの削除：管理者は、ユーザー/内部/アクション/ユーザーを削除を使用して、ユーザーを手動で削除できます。
* ユーザーの消去：削除後、管理者はユーザーレコードを完全に消去して、データ保持ポリシーに準拠し、古いユーザーデータへの不正アクセスを防止できます。

詳しくは、以下を参照してください。

* [ユーザーおよびユーザーグループの追加](https://experienceleague.adobe.com/ja/docs/learning-manager/using/admin/user-management/add-users-user-groups)
* [ユーザーを消去](https://experienceleague.adobe.com/ja/docs/learning-manager/using/admin/purge-users)

## 管理口座の廃止

管理アクセス権が不要になった場合は、それを削除する必要があります。 管理アカウントの廃止には、以下が含まれる場合があります。

* ユーザーからの管理者ロールの取り消し
* 完全な管理アクセスが不要になった場合は、権限を減らします。
* 必要に応じて、システムからユーザーを削除します。

不必要な管理アクセスを定期的に確認して削除することで、最も権限の少ないアクセスを維持できます。

### システム管理者権限の削除(Admin Console)

システム管理者が組織を離脱したりロールを変更したりした場合、権限は速やかに取り消す必要があります。

1. システム管理者としてAdobe Admin Consoleにログインします。
2. ユーザー/管理者に移動します。
3. 削除する管理者を選択します。
4. 詳細オプション(...)アイコンをクリックして、管理者を編集し、システム管理者の役割を削除するか、「ユーザーを削除」を選択して、ユーザーを組織から完全に削除します。
5. 管理者が他の役割（製品管理者、サポート管理者など）を持っていた場合は、それらの役割も取り消します。

>[!IMPORTANT]
>
>出国管理者が組織の契約所有者である場合、その契約所有者の役割を別のユーザーに転送してからでなければ、そのユーザーを削除できません。 必要に応じてAdobeサポートにお問い合わせください。

詳しくは、以下を参照してください。

* [Admin Console上のユーザーアカウントを作成、更新、または削除する](https://helpx.adobe.com/jp/enterprise/using/manage-users-individually.html)
* [組織所有のアカウントから離脱する方法](https://helpx.adobe.com/jp/enterprise/using/leave-organization.html)

### ALM管理者ロールの削除

ユーザーのアカウントを削除せずにALM管理者のアクセス権を取り消すには（ユーザーが学習者のままである場合など）:

1. Adobe Learning Managerに管理者としてログインします。
2. ユーザー/社内に移動します。
3. ユーザーを検索して選択します。
4. アクション/役割を削除/管理者を削除を選択します。

ユーザーが学習者ロールに戻ります。 それらの学習履歴とコース登録は保持されます。

詳細については、[ユーザーおよびユーザーグループの追加](https://experienceleague.adobe.com/ja/docs/learning-manager/using/admin/user-management/add-users-user-groups)を参照してください。

### ユーザーの削除と消去

ユーザーが組織を完全に離脱し、プラットフォームからアカウントを削除する必要がある場合：

* ユーザーを削除します。ユーザー/内部/ユーザー/アクション/ユーザーを削除を選択します。 これにより、アカウントが無効になり、アクティブなアクセスが削除されます。
* ユーザーを消去します。削除後、ユーザー/ユーザーのクリーンアップに移動し、削除月を選択します。次に、ユーザーを選択して、「アクション/ユーザーを消去」を選択します。 完全に削除すると、すべてのユーザーレコードが削除されます。

詳細については、[ユーザーの消去](https://experienceleague.adobe.com/ja/docs/learning-manager/using/admin/purge-users)を参照してください。


## セキュリティと責任の共有

Adobe Learning Managerは、共有責任モデルに基づいて動作します。

* Adobeは、基盤となるALMプラットフォームとインフラストラクチャのセキュリティを確保する責任を負います。
* お客様は、ALMアカウント内の管理アクセス、役割の割り当て、ユーザーライフサイクルアクティビティを管理する責任があります。

Adobe Learning Managerのセキュリティ対策の詳細については、[Adobe Learning Managerセキュリティの概要(PDF)](https://experienceleague.adobe.com/docs/learning-manager/assets/alm-security-whitepaper-2024.pdf?lang=ja)を参照してください

## 拡張セキュリティ機能の範囲

### SCG-ENH-CMP

Adobeは、文書化されたコンポーネントのインベントリ、所有権、およびライフサイクル管理プロセスを維持し、システムコンポーネント全体で管理された設定とコンプライアンスを確保します。

### SCG-ENH-API

Adobeは、標準化されたAPIセキュリティ制御（認証、許可、モニタリングなど）を実施します。このセキュリティ制御は、文書化されたガバナンスおよびプラットフォーム保護対策によってサポートされています。

### SCG-ENH-MRG

Adobeは、レビューや承認管理を含む正式な変更およびマージ管理プロセスを適用して、システムの整合性を維持し、展開に関するリスクを軽減します。

### SCG-ENH-VRH

Adobeは、定義済みの脆弱性管理と修復プロセスに従って行われ、特定、優先順位づけ、トラッキング、およびタイムリーな解決が行われます。
